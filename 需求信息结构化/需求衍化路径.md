## 需求衍化路径
```markdown

┌────────────────────┐
│     Business Needs │（组织/业务目标）
└────────┬───────────┘
         │  转化为战略级目标
         ▼
┌────────────────────┐
│        📄 BRS      │ Business Requirements Specification  
│  - 业务动机、流程、政策、环境  │
└────────┬───────────┘
         │  抽取相关涉众需求
         ▼
┌────────────────────┐
│       📄 StRS      │ Stakeholder Requirements Specification  
│  - 用户/干系人的期望与约束    │
└────────┬───────────┘
         │  转化为系统级可实现项
         ▼
┌────────────────────┐
│       📄 SyRS      │ System Requirements Specification  
│  - 系统“应该做什么”           │
└────────┬───────────┘
         │  拆分为软件实现细节
         ▼
┌────────────────────┐
│       📄 SRS       │ Software Requirements Specification  
│  - 模块、接口、数据库、约束   │
└────────────────────┘
```
