## 1. 背景与范围

【系统介绍】

- **目标与定位**：月台管理系统为物流园区、工业厂区等场景中常见的子系统，核心任务为通过一定的规则有序地分配火车车辆停靠制定的月台进行装卸货作业，并完成货物核验。（月台指的是仓库前火车停靠的平台，类似车位）。
- **服务对象/场景**：物流园区、工业厂区等。
- **覆盖范围（包含/不包含）**：多仓库园区、厂区均有需求，不涉及对接第三方。
- **关键差异化**：待补充。

【角色与目标】

- **司机**：
  - 目标1：预约申请入园，通过申请后在申请的时间段入园。
  - 目标2：无需与安保人员沟通，便利入园，然后进入排队队列。
  - 目标3：知晓可进入月台作业的时间，进入月台，供装卸作业人员开展月台作业。
  - 目标4：货物全部核验后离园，货物未核验完成不可离园。
- **月台管理员**：
  - 目标1：审核司机入园申请。
  - 目标2：根据系统自动分配的月台作业车辆信息，以及月台实时情况/突发情况，进行车辆叫号，通知排队车辆前往月台作业。
  - 目标3：可以提前设置车辆行驶耗时阈值、超时通知次数、通知响应时间，以让系统判断车辆是否到达超时。在车辆到达超时时，无需手动剔除无效车辆，知晓此时可以叫下一辆车。
  - 目标4：线上登记实际接收货物类型和对应数量。
- **客户管理员**：
  - 目标1：核验货物登记信息，避免后续货物纠纷。
- **园区管理员**：
  - 目标1：设定功能。
  - 目标2：在货物有争议时复核。

## 3. 业务流程（AS‑IS 与 TO‑BE，对比与改进点）

1. `PROC-PLT-001` **入园预约与排队**
   - **AS-IS**（无月台管理或传统方式）：司机到园区现场排队，人工核验，沟通成本高。
   - **TO-BE**（使用本系统后）：司机提前线上预约，系统审核并分配时段，入园后自动进入虚拟排队。
   - **解决的问题/改进点**：缩短等待时间，减少人工核验错误。（关联 KPI-PLT-001）
   - **主要参与角色**：司机、月台管理员。
2. `PROC-PLT-002` **月台分配与叫号**
   - **AS-IS**：人工调度月台，信息滞后，月台可能空置或拥堵。
   - **TO-BE**：系统根据规则自动分配月台，管理员可动态叫号，车辆按序入场。
   - **解决的问题/改进点**：提升月台利用率，降低人工干预。（关联 KPI-PLT-002）
   - **主要参与角色**：月台管理员、司机。
3. `PROC-PLT-003` **货物核验与登记**
   - **AS-IS**：纸质单据，后续争议难追溯。
   - **TO-BE**：线上登记货物类型与数量，客户管理员与园区管理员可复核。
   - **解决的问题/改进点**：减少货物纠纷，提升数据可追溯性。（关联 KPI-PLT-003）
   - **主要参与角色**：司机、客户管理员、园区管理员。
4. `PROC-PLT-004` **超时处理与车辆剔除**
   - **AS-IS**：人工观察车辆是否到位，容易延误。
   - **TO-BE**：系统自动检测车辆到位耗时，超时自动剔除并通知下一辆车。
   - **解决的问题/改进点**：提高调度效率，降低链路中断风险。（关联 KPI-PLT-004）
   - **主要参与角色**：月台管理员。

## 4. 用户故事

- **US-PLT-001 入园预约与排队**\
  As a **司机**, I want to **线上预约入园并自动进入排队**, so that **我能减少等待时间并顺利入园**。

  - **前置条件**：司机已在系统中注册并提交车辆信息。
  - **基本流程**：司机发起预约 → 管理员审核 → 系统分配时段 → 入园后进入虚拟排队。
  - **验收标准（清单）**：
    - 司机能提交货物清单。
    - 月台管理员能登记货物收货信息。
    - 客户管理员能核验货物数据。
    - 园区管理员能在争议时复核。

- **US-PLT-002 月台分配与叫号**\
  As a **月台管理员**, I want to **根据系统分配和实时情况叫号**, so that **车辆能有序进入月台作业**。

  - **前置条件**：系统已存在等待车辆队列。
  - **基本流程**：系统初始分配月台 → 管理员选择并叫号 → 司机收到通知 → 车辆进场作业。
  - **验收标准（清单）**：
    - 系统能自动分配月台给车辆。
    - 管理员可手动调整或叫号。
    - 司机能收到叫号通知。
    - 车辆能在规定时间到达月台。

- **US-PLT-003 货物核验与登记**\
  As a **客户管理员**, I want to **在线核验和登记货物信息**, so that **能减少货物纠纷**。

  - **前置条件**：司机已提交货物清单，车辆已到达月台。
  - **基本流程**：司机提交货物清单 → 月台管理员登记 → 客户管理员核验 → 园区管理员复核（有争议时）。
  - **验收标准（清单）**：
    - 司机能提交货物清单。
    - 月台管理员能登记货物收货信息。
    - 客户管理员能核验货物数据。
    - 园区管理员能在争议时复核。

- **US-PLT-004 超时处理与车辆剔除**\
  As a **月台管理员**, I want to **系统自动检测车辆是否超时并剔除**, so that **我能保持调度链路顺畅**。

  - **前置条件**：车辆已被叫号，但未按时到达。
  - **基本流程**：系统监测车辆到达耗时 → 超时则自动剔除并通知下一辆车 → 管理员收到通知。
  - **验收标准（清单）**：
    - 系统能检测车辆到达耗时。
    - 超时后车辆能自动剔除。
    - 下一辆车能被及时叫号。
    - 管理员收到剔除通知。
